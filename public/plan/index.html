SpawnHero RPG

What you can do in this game, and what an RPG is.

How do you win or lose?

The game ends when some sort of win condition is met. It's up to you to determine what the win condition is. Some examples are:

  • The player wins the game when all coins on the map are collected.
  • The player wins when all quests are completed.
  • The player wins when all treasure is found.
  • The player wins when all enemies have been defeated.
  • The player wins when all enemies have been evaded.
  • The player wins when a riddle is solved.

Types of Objects

When planning your game map, it is also important to know what types of objects can be placed on the screen. Note that just because an object is called 'enemy', it does not have to look like an enemy or behave like an enemy. There are many creative ways to use these objects to make them look & behave in creative ways.

  • hero: properties
  • enemy: properties / If a player touches an enemy, the game is over.
  • object: properties
  • collectable: properties
  • interactable: properties
  • freeze: If a player touches freeze, they will be frozen for the time indicated in the settings.
  • slow: If a player touches slow, they will be slowed down for the time indicated in the settings.
  • speed: If a player touches speed, they will speed up for the time indicated in the settings.

It is critical to review the SpawnHero RPG Docs for descriptions of how each element affects game play, and to learn the rules and constraints of the system. This will allow you to design a level that is actually possible to build with the tools that are available.

SpawnHero RPG Prototype

Before you start programming, it is important to create a prototype for your game level. Your prototype is a map for building the game, so you can focus on making the game challenging and fun, without worrying about the code.

This game has a map size of 46 by 34 tiles. Each block is 40px by 40px, making the total map size 1840px by 1360px.
